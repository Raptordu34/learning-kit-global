Callouts

Conseil : Préférer EXPLAIN ANALYZE à EXPLAIN seul — le premier exécute réellement la requête et retourne les temps mesurés.
Info : PostgreSQL utilise un planificateur basé sur des statistiques collectées par ANALYZE. Relancer ANALYZE après une grande insertion peut changer significativement le plan d'exécution.
Attention : REFRESH MATERIALIZED VIEW sans CONCURRENTLY pose un verrou exclusif — toutes les lectures sont bloquées pendant la durée du rafraîchissement.
Danger : Ne jamais lancer DROP TABLE en production sans sauvegarde préalable. L'opération est irréversible et instantanée.

Code

Bloc multi-lignes — pre

-- Vue matérialisée avec index
CREATE MATERIALIZED VIEW ventes_mensuelles AS
    SELECT
        DATE_TRUNC('month', date_vente) AS mois,
        region,
        SUM(montant)                    AS total
    FROM ventes
    GROUP BY 1, 2
WITH DATA;

CREATE INDEX ON ventes_mensuelles (mois, region);
